## node-pty caveat (read before "fixing")

`node-pty@1.1.0`'s npm tarball is missing the `deps/winpty/shared/` submodule, so a Windows source rebuild fails with `'GetCommitHash.bat' is not recognized`. The package ships NAPI prebuilds in `prebuilds/win32-x64/` that load fine on Electron 41 (NAPI is ABI-stable across Electron majors), and node-pty's loader (`lib/utils.js`) falls back to those when `build/Release/*.node` is absent. We deliberately exclude node-pty from `electron-rebuild` via `--only=keytar` and disable electron-builder's rebuild via `"npmRebuild": false`. Do not "fix" by trying to source-build node-pty without first checking whether a newer node-pty release ships the winpty submodule.

## Goals & non-goals

**Goal:** clear the remaining audit advisories by completing three coupled major upgrades, with full functional verification and a clean rollback path.

**Non-goals:** rewriting the IPC surface, switching state management, or modernizing the UI. Each phase is upgrade-only — no opportunistic refactoring.

## Recommended ordering

1. **Vite 5 → 7** — renderer-only, doesn't touch the running Electron runtime. Lowest blast radius; ship first to retire the CJS warning.
2. **electron-builder 25 → 26 + Electron 33 → 41** — done together because the builder must know about the new Electron version and rebuild the native modules.

Vite 7 (not 8) chosen because `@vitejs/plugin-react@5` peers with Vite 4–8, while `@vitejs/plugin-react@6` requires Vite 8 only. Stay one major back from bleeding edge.

Don't bundle phases. Two PRs so a regression in Phase 2 doesn't force reverting Phase 1's work.

## Phase 1 — Vite 5 → 7

### What changes

- `package.json`: add `"type": "module"`. All `.js` at the project root now ESM by default.
- `dist/main/package.json` and `dist/preload/package.json`: tiny `{ "type": "commonjs" }` files generated by a postbuild step (because `dist/` is gitignored). Override the root setting so `tsc`'s CommonJS output continues to load under Electron's CJS main.
- `scripts/write-cjs-markers.mjs`: new build helper.
- `package.json` scripts: `build:main` runs the marker writer after `tsc`.
- `postcss.config.js` and `tailwind.config.js`: no change — already `export default`.
- `vite.config.ts`: no change — Vite handles TS configs.
- Bumps: `vite@^7`, `@vitejs/plugin-react@^5`.

### Risks

- Native module rebuild is NOT triggered by Vite. `node-pty` and `keytar` untouched.
- Any future root-level `.js` file using `require(...)` would break. Currently none.
- Verify nothing in the packaged app reads `package.json` and chokes on the new `"type"` field (should be safe — Electron's main loader respects it).

### Verification

1. `npm run build:main` — main compiles, CJS markers written.
2. `npx vite build` — renderer builds, no CJS warning.
3. `npm run dev` — app starts; renderer hot-reloads.
4. `npm run dist` — installer is produced, installs cleanly.

### Rollback

Single-commit phase. `git revert` and you're back on Vite 5.

## Phase 2a — Pre-flight for the Electron upgrade

Before touching `electron`:

- **`keytar` is the high-risk dependency.** Officially maintenance-only. Before bumping Electron:
  1. Try the rebuild. If it succeeds, proceed.
  2. If it fails: replace with **`@napi-rs/keyring`**. API parity with our usage (`getPassword` / `setPassword` / `deletePassword` keyed on service+account). Single file change in `src/main/services/credentialStorage.ts`. Do this as a separate commit *before* bumping Electron so it can be reverted independently.
  3. Last resort: migrate to Electron's built-in `safeStorage`. Bigger refactor — manage our own ciphertext file. Out of scope.
- **`node-pty@^1.1.0`** is actively maintained. Should rebuild cleanly. If not, bump to latest.
- **`electron-updater@^6.3.9`** supports Electron 30+. Should be fine through 41.
- **`@electron/rebuild`**: pulled in by electron-builder 26. Verify it succeeds against `keytar` and `node-pty` on Windows.
- **Snapshot a working installer** before starting (current `release/AWS Profile Manager Setup x.x.x.exe`) for comparison.

## Phase 2b — Electron 33 → 41 + electron-builder 25 → 26

### Steps

1. `npm install electron@^41 electron-builder@^26 --save-dev`.
2. `npx electron-rebuild` (or trust `electron-builder install-app-deps` during `dist`). Verify `keytar` and `node-pty` rebuild against the new ABI.
3. Run `npm run dev`. Triage deprecation warnings.

### Manual regression checklist

Run on a clean install (not just `npm run dev`):

- [ ] Launch from installer; splash → main window.
- [ ] Master password create / unlock / forget.
- [ ] Add/edit/delete profile.
- [ ] Manual refresh on a profile (full SAML round-trip).
- [ ] Refresh-all (single credential prompt → multi-profile refresh).
- [ ] Auto-refresh fires after expiry-threshold.
- [ ] Auto-refresh **not** paused when offline (the bug just fixed — disconnect Wi-Fi/dock and watch).
- [ ] Tray menu: Open, Refresh All, Pause/Resume, Exit.
- [ ] Embedded terminal opens and accepts input (`node-pty` ABI rebuild test).
- [ ] AI Assistant returns a CLI suggestion (if Open WebUI configured).
- [ ] Backup config → restore config round-trip.
- [ ] Auto-updater: build a `v1.2.4`, install `v1.2.3` first, confirm in-app update detects and installs. **Real machine, not dev box.**

### Risks

- **Keytar rebuild failure.** Mitigation: `@napi-rs/keyring` swap-in.
- **Auto-updater format change in electron-builder 26.** Critical: existing v1.2.3 installs must be able to update *to* the new version. Test this path explicitly. If old clients can't read the new manifest, you've stranded existing users.
- **SmartScreen / AV reputation.** New executable will trigger fresh reputation checks. Expect "Unblock-File" support questions for a few days.
- **Node 22 in Electron 41.** Quick grep for deprecated APIs (`Buffer` constructor, `url.parse`, etc.) before the upgrade.

### Rollback

- Phase 2 on its own branch; tag `pre-electron-41` before merging.
- If post-release regression: revert the branch, ship `v1.2.x-rollback` from the tag. Existing users updating from `v1.2.3` should still receive it normally (we never changed the manifest format on rollback).
